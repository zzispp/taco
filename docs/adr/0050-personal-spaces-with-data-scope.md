# Use personal asset spaces with Data Scope management

Each user owns one Personal Asset Space containing their Managed Files and Logical Folders; there is no default global shared directory or per-user sharing in the first release. File operations require both the capability permission and the caller's Data Scope: `SelfOnly` limits a user to their own space, department scopes cover spaces owned by users in the permitted departments, and `All` covers every space. An Asset Management View may aggregate authorized spaces for managers without changing ownership. Soft-deleting a user preserves an Archived Personal Asset Space for scoped administration; no automatic ownership transfer occurs. This reuses Taco's existing RBAC data-scope semantics and avoids inventing a second file-specific authorization model.
