# Reconcile upload completion through a durable state machine

Upload Sessions persist `Open`, `Completing`, `Completed`, `Aborted`, and `Expired` states. Completion claims the `Completing` state atomically, so a concurrent caller receives an explicit in-progress result instead of assembling the same object. Provider completion, status, and deletion operations are idempotent; only `Completed` sessions expose a Managed File. Deterministic verification failures, including size or digest mismatches, abort the provider session and terminalize the database session so its reservation is released. A cleanup task reconciles sessions stuck in `Completing` after process or database failure and records an explicit terminal outcome, rather than guessing whether a Provider object was committed.
